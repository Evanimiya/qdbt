# QDBT 추출 아키텍처 재설계 — 코드 기반 추출

작성일: 2026-06-25
상태: 설계 확정 (목업 검증). 미구현.
배경: LLM 기반 추출의 path 꼬임 문제 → 코드 기반으로 전환 결정.

---

## 0. 왜 바꾸나 — LLM 추출의 한계

현재: 엑셀 전체를 LLM에 넘겨 LLM이 path까지 생성.
문제 (실데이터 진단으로 확인):
- "재료비 > 기구부 > 차폐 > Maint Door"가 "기구부 > 차폐"로 (대분류 누락)
- "인건비 > 인건비(협력사site) > 설계"가 "인건비 > 설계"로 (중간 누락)
- 구분자 혼용 (>와 /)
- depth 미기록
근본 원인: 구조화된 데이터(엑셀 셀→DB행)를 LLM에 맡김.
  엑셀은 셀 위치가 명확한데 LLM이 매번 다르게 추측 → 부정확/느림/비쌈.

핵심 통찰 (사용자):
  "엑셀 읽어 DB로 추출하는 건 LLM이 아니라 코드가 정확하다.
   LLM은 ①빈칸 채우기 ②클러스터링 ③카탈로그 매칭에만."

---

## 1. 역할 분담 (재정의)

### 코드가 할 일 (결정론적)
- 셀 읽기 → DB 행
- 셀 병합 풀기 (상위 분류를 빈 칸에 상속)
- 분류 계층 path 구성 (대>중>소를 열에서 조립)
- 숫자 파싱 (단가/수량/금액)
- 헤더/소계 행 구분

### LLM이 할 일 (의미·판단)
- 헤더 인식 → 열 역할 제안 (1회, 사람이 확인)
- 클러스터링 (같은 물건 묶기)
- 카탈로그 매칭 (표준품목 연결)
- 비교 단위 깊이 제안 (카탈로그 참조)

→ LLM은 "양식 파악 1회"만. 수백 행 추출은 코드.

---

## 2. 2단계 추출 방식

### 1단계: 헤더 인식 (LLM 제안 + 사람 확인)
- LLM이 상위 행을 보고 열 역할 제안:
  "C=대분류, D=중분류, E=소분류, F=품목, I=단가..."
- 빈 분류 레벨 감지 (예: "재료비"가 헤더 빈 칸인데 대분류 위치)
- 사람이 인터랙티브 화면에서 확인/수정 (핵심)

### 2단계: 추출 (코드, 결정론적)
- 확인된 열 매핑으로 셀 → DB
- 병합 풀기 + 빈칸 상속
- path 구성 (지정된 분류 열 조립, 구분자 " > " 통일)
- 분류(대/중/소/세)와 정보(품목/규격/단가)를 역할로 분리

---

## 3. 화면 설계 (목업 확정)

docs/qdbt_combined_mockup.jsx — 통합 화면 (확정)

### 구조: 엑셀 표 하나 + 두 탭
**①탭 열 역할 지정**
- 엑셀을 그대로 표시 (열문자 A,B,C..., 행번호 R1,R2...)
- 각 열 위 드롭다운: 대/중/소/세분류, 품목명/규격/수량/단위/단가/금액/비고/무시
- LLM 제안값 미리 채움, 사람이 확인/수정
- 헤더 행 지정 (그 아래부터 데이터)
- 빈 분류 레벨 경고 (재료비 케이스)
- 분류=파랑, 정보=초록/빨강 색 구분

**②탭 비교 단위 선택**
- 같은 표에서 분류 칸을 클릭 → 비교 단위 지정
- "기구부" 클릭=중분류 단위, "차폐" 클릭=소분류 단위
- 일괄 선택 바: [대분류로 일괄][중분류로 일괄][소분류로 일괄][전체 해제]
  → 일괄로 깔고 개별 클릭으로 조정
- 같은 값은 첫 행만 표시 (병합처럼)
- 아래 비교 단위별 집계 (금액 합계)

### 트리 vs 클릭
- 분류가 열로 분리되니 트리 펼침 불필요.
- 클릭이 직관적 (차폐 클릭 한 번 = 소분류 단위).
- 기존 트리 UI(qdbt_tree_mockup)를 이 방식이 대체.

## 4. 미정 사항 (구현 시 결정)
1. 비교 단위 선택 제약: 같은 줄기 충돌 처리
   (기구부 선택 + 차폐 선택 동시 → 더 구체적(차폐) 우선이 현재 동작.
    상위 자동 해제할지 미정)
2. 일괄 선택이 기존 선택 대체(현재) vs 추가
3. 빈 분류 레벨(재료비) 표현 방식 (B열을 대분류로 vs 별도 표시)
4. 여러 행 헤더, 특정 행 제외, 단가×수량 검산

---

## 5. 구현 단계 (큰 재설계)
1. 코드 추출 엔진: 열 매핑 → 셀 → path/DB (결정론적)
   - 병합 풀기(build_merge_fill_map 활용), 빈칸 상속
   - path 구성 (구분자 통일)
2. 헤더 인식 LLM: 상위 행 → 열 역할 제안
3. 인터랙티브 화면: combined_mockup 구현
   - 엑셀 표 렌더, 열 드롭다운, 분류 클릭, 일괄 선택
4. 기존 LLM 추출 → 신규 코드 추출로 교체
5. 비교 단위(units)를 compare_units로 저장 (기존 연결)

### 의존
- parse_xlsx에 이미 있음: 병합복원, 헤더/중분류 탐지
  → 절반은 와 있음. 열 매핑 명시 + 코드 path 구성만 추가.
- 기존 트리/compare_units 저장 구조 재활용 가능.

---

## 6. 영향
- 추출이 빨라짐 (행 추출에서 LLM API 사라짐).
- path 정확 (코드 결정론적, 추측 없음).
- 양식 차이는 ①탭에서 사람이 1회 흡수.
- LLM 비용 절감 (양식 파악 1회만).

### 주의
- 양식이 유사하나 세부(병합/채움) 다름 → 열 매핑으로 흡수.
- 헤더 인식이 틀려도 사람이 ①탭에서 교정 → 안전.
