## 이번(오늘) 작업 룩백

### ★ G. 비교 단위 (group by) 1단계 — NEW v0.8.9
견적서를 분류 레벨(대/중/소/세부)로 묶어 금액 합계 표시.
파워피벗처럼 레벨 변경 가능. 데이터(잎)는 보존, 표시만 묶음.
- 견적서 상세에 "📊 비교 단위" 섹션 (레벨 드롭다운)
- 묶음별 금액 합계 + 트리 펼침 (세부+줄번호)
- 레벨을 submissions.compare_level에 저장
  → 업체별 레벨 지정의 토대 (A=소분류, B=중분류 가능)
- group_items_by_level(): path를 레벨까지 잘라 group by
  path 없으면 category>name 폴백 (안전)
설계 근거: "A의 소분류 = B의 중분류"일 수 있어
  업체마다 비교 레벨을 사용자가 지정해야 함. 기본값 중분류.


### A. 엑셀 시트 선택
업로드 시 시트 여러 개면 탭 체크박스 → 선택 시트만 추출.
불필요 시트(일정 등) 제외.
※ 재추출(상세 화면)에도 UI 추가했으나 청킹 경로에서 미작동 → 알려진 이슈

### B. 병합 셀 복원
대분류/중분류 세로병합, Price 2줄헤더를 파서가 풀어
병합 영역 전체에 대표값 채움 → 각 품목이 대/중/소분류를 가짐.

### C. 추출 중지 + 진단
- 멈췄을 때 ⏹ 중지 버튼 (30초 후 힌트)
- 워커/파이프라인 단계 로그 (어디서 멈추는지 터미널 출력)

### D. DB 저장 안전화
"1,000원" 문자열 단가도 숫자 변환. 실패 항목 콘솔 로그.

### E. LLM 타임아웃
180초 응답 한도. 무한 대기 방지 → 시간 초과 시 failed.

### F. ★ 대용량 청킹 (핵심)
문제: 병합 복원으로 분류 채운 파싱 결과가 20만 자. LLM이 못 삼켜 멈춤.
해결: 중분류 단위로 조각내 추출 → 합침.
- 헤더 "중분류" 글자 탐지 → 그 열로 묶음 (폴백 30행 분할)
- 30행 한도, 초과 시 재분할
- 조각별 추출 → items 합침
- "추출 중 N/M" 실제 진행률
실측: 24조각, 조각당 ~30초, 완주 성공.

## 알려진 이슈 (다음 작업에서 반영)

### 1. 재추출 시 시트 선택 미작동
업로드 화면은 OK. 재추출(상세)은 모든 탭 추출됨.
→ 청킹 경로에서 sheet_names 전달 점검 필요.

### 2. 합계 행(SUB-TOTAL) 오추출
중간 합계를 품목으로 인식.
→ 추출 단계 거르기(프롬프트/후처리) + 오추출 항목
   사용자가 삭제/비활성화하는 기능 필요.

### 3. 줄 번호(행 번호) 표시
리뷰 편의용. 원본 몇 행인지 추출 결과에 포함.
(클러스터링엔 불필요). submission_items.line_no 컬럼 활용.

### 4. Special nego 부호 (차감 항목이 +로 표기)
총액에서 빼주는 special nego가 견적서엔 +로 적힘 → 추출도 +.
실제로는 -여야 함. 사용자가 부호 수정 가능해야.

### 5. 추출된 이름의 들여쓰기/작은폰트 (원인 추정)
일부 항목 이름이 폰트 작고 bold 아니고 들여쓰기됨.
추정 원인: 파서 cell_to_str가 셀 앞 공백을 [indent=N] 태그로
보존 → LLM이 이름에 반영, _strip_indent_prefix로 완전히
안 떼진 잔재로 보임. → 실제 이름값 확인 후 파서/정리 수정 필요.

### ★ 2·4·5 묶음: "추출 결과 편집 기능"
합계행 삭제, special nego 부호수정, 이름 정리가 모두
"사용자가 추출 결과를 손보는" 편집 기능으로 묶임.
항목 삭제/비활성화 + 금액·부호 수정 + 이름 정리를 한 화면에서.
→ 1·2번(비교단위 선택 화면)과 함께 설계 권장.

## [추가] 비교 단위 트리 — 전체 설계 확정 (2026-06-25)

상세: docs/QDBT_비교단위트리_전체설계서.md

### 확정된 큰 그림
1. **파워피벗 모델**: 데이터(잎) 보존, 비교 단위는 "보기 방식".
2. **트리 비교 단위**: +/− 펼침, "더 펼칠 수 없는 노드"=비교 단위.
   그룹별로 다른 깊이 가능 (기구부는 소분류, 이윤은 대분류).
3. **재클러스터링**: 비교 단위 묶음을 클러스터링 input으로 교체.
   어느 레벨로든 자유. (group_items_by_level의 일반화)
4. **LLM 레벨 제안**: 업체별 적절 비교 단위 제안. 기존 카탈로그 참조.
   "A소분류=B중분류" 해결. 제안→검토→확정.
5. **★ 계층 네 군데 전파**: 추출/비교/클러스터/카탈로그 일관.
   카탈로그도 트리 (잎까지 보존 → 레벨 재조정·시계열 리콜).

### DB 보강 필요 (구현 시 먼저!)
현재: submission_items만 path 견고. 나머지는 계층 미연결.
보강:
- submissions + compare_units (JSON, 그룹별 깊이)
- catalog_items + canonical_path, depth, observed_paths(다중경로)
- catalog_clusters + compare_unit_path, compare_level
- price_history + item_path, compare_unit  ← ★ 처음부터 필수 (소급불가!)

### 구현 순서 (수정됨 — DB 먼저)
1. DB 스키마 확장 (계층 전파 컬럼)
2. 추출 시 path를 클러스터·카탈로그·price_history로 전파
3. 그룹별 트리 UI (목업 qdbt_tree_mockup.jsx 검증됨)
4. 비교 단위 재클러스터링
5. 업체 간 비교 화면
6. LLM 레벨 제안 (카탈로그 참조)

### 주의
- price_history.item_path는 가격 기록 시점에 남겨야 함. 소급 불가.
- 계층 전파는 데이터 쌓인 뒤 마이그레이션 부담 → DB 먼저.

## [v0.9.3] nego 수기 입력 + 이름 충돌 구분 (2026-06-25)

### Special Nego 수기 입력 (별도 항목)
nego가 항목마다 제각각 → 자동 추출 대신 사람이 수기 입력.
- nego = submission_items 항목 (is_nego=1)
- ②탭 비교 단위 트리 위에 입력 패널 (항목명+금액)
- 양수 입력 → 자동 음수(-abs) 저장
- 합계·클러스터링·비교에 자동 포함 (submission_items이므로)
- 재추출 시 보존 (delete_submission_items(keep_nego=True))
- 추가/수정/삭제 가능 (nego/add, nego/<id>/update, list)
- is_nego 컬럼 마이그레이션
검증: 추가(-3M)→공급가액(11.6M)→재추출 보존→수정(8.6M)
